# Automatização de Servidores Linux com Ansible: Integração Sem Agente
> **TL;DR:** Ansible automatiza configurações via SSH, sem agente, com integração nativa a nuvem e CI/CD.
## O que é e por que usar
Ansible é uma ferramenta de automação de infraestrutura que utiliza SSH para gerenciar servidores sem a necessidade de instalar agentes nos nós remotos. Sua arquitetura baseada em módulos idempotentes e playbooks em YAML permite automatizar desde a instalação de pacotes até a orquestração de aplicações. Para integração, Ansible se destaca pela compatibilidade nativa com provedores de nuvem (via plugins de inventário dinâmico) e ferramentas como Terraform, Docker e Jenkins, facilitando a implementação de infraestrutura como código (IaC) e pipelines de CI/CD.
## Requisitos
Não há requisitos oficiais específicos de hardware, pois o Ansible é leve e roda em qualquer sistema com Python. Contudo, para controlar N servidores remotos, são necessários:
- Controlador: Python 2.7+ (padrão em Linux) e Ansible instalado.
- Servidores remotos: SSH habilitado e acesso via chave SSH (sem agente). Python é opcional para módulos básicos, mas recomendado para funcionalidades avançadas.
- Rede: Conectividade na porta 22 entre controlador e servidores.
## Instalação
### Método 1: Via repositório (Debian/Ubuntu)
```bash
sudo apt update
sudo apt install -y ansible
```
### Método 2: Via repositório (RHEL/CentOS)
```bash
sudo yum install -y ansible
```
## Configuração
Crie um arquivo de configuração `ansible.cfg` no diretório do projeto:
```ini
[defaults]
inventory = inventory.ini
remote_user = ubuntu
private_key_file = ~/.ssh/id_rsa
```
## Exemplo Prático
### Cenário: Instalar e iniciar o Apache em múltiplos servidores
1. **Criar arquivo de inventário** (`inventory.ini`):
   ```ini
   [servidores]
   servidor1 ansible_host=192.168.1.10
   servidor2 ansible_host=192.168.1.11
   ```
2. **Criar playbook** (`instala_apache.yml`):
   ```yaml
   ---
   - name: Instalar Apache e iniciar serviço
     hosts: servidores
     become: yes
     tasks:
       - name: Instalar Apache
         ansible.builtin.apt:
           name: apache2
           state: present
       - name: Iniciar serviço
         ansible.builtin.service:
           name: apache2
           state: started
           enabled: yes
   ```
3. **Executar playbook**:
   ```bash
   ansible-playbook -i inventory.ini instala_apache.yml
   ```
### Testando o Playbook
Antes de aplicar as mudanças, é possível testar o playbook usando os seguintes modos:
- Modo de simulação (`--check`): Executa o playbook sem aplicar alterações, mostrando o que seria feito.
  ```bash
  ansible-playbook -i inventory.ini instala_apache.yml --check
  ```
- Modo diff (`--diff`): Mostra as diferenças que seriam aplicadas nos arquivos.
  ```bash
  ansible-playbook -i inventory.ini instala_apache.yml --check --diff
  ```
- Modo verboso (`-v`): Fornece mais detalhes durante a execução.
  ```bash
  ansible-playbook -i inventory.ini instala_apache.yml -v
  ```
## Armadilhas Comuns
### ⚠ Falha de conexão SSH (Código 255)
**Sintoma:** Ansible falha ao conectar ao servidor remoto com erro de código 255.
**Causa:** Credenciais incorretas, chave SSH sem permissão 600, ou firewall bloqueando porta 22.
**Solução:**
```bash
chmod 600 ~/.ssh/id_rsa
```
### ⚠ Erro de verificação de host key
**Sintoma:** Erro "invalid argument" ao conectar ao servidor.
**Causa:** Chave do servidor não está em `~/.ssh/known_hosts`.
**Solução:**
```bash
ssh-keyscan -H servidor >> ~/.ssh/known_hosts
```
### ⚠ Inventário desatualizado
**Sintoma:** Falha ao executar playbook com mensagens de host desconhecido.
**Causa:** Hostnames ou IPs incorretos no arquivo `inventory.ini`.
**Solução:**
```bash
ansible all -i inventory.ini -m ping
```
## Dicas de Otimização
- Use inventário dinâmico para AWS: configure `aws_ec2.yml` com plugin `aws_ec2` e execute `ansible-inventory -i aws_ec2.yml --list`.
- Aumente o paralelismo adicionando em `ansible.cfg`: `forks = 50`.
- Trate erros com blocos `rescue` em playbooks para reverter alterações em caso de falha.
## Conclusão
Ansible oferece uma solução robusta para automação de servidores Linux sem agente, com foco em integração através de suporte nativo a nuvem, Docker e CI/CD. Seus principais pontos fracos são a dependência de SSH e a curva de aprendizado para playbooks complexos. Para integração unificada, combine com Terraform no pipeline de IaC.
